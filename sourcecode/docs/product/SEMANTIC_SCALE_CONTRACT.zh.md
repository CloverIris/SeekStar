# SeekStar 连续多尺度语义契约

状态：权威（Normative）  
适用范围：L0–L3 的生成、世界池、投影、交互、设计与验收  
上位契约：[SeekStar 产品契约](./PRODUCT_CONTRACT.zh.md)

## 1. 模型定义

SeekStar 的 L0–L3 是同一张连续语义世界在不同观察尺度下的投影，正式模型为：

> **共享语义地图金字塔：可变密度、多对多 refinement、连续 XY、来源验证封顶。**

它不是目录树、固定四层列表、词云，也不是盒子套盒子。改变尺度不会进入另一个坐标系；它只改变当前世界中哪些对象、关系、摘要和来源细节足够重要而应被看见。

## 2. 基本概念

### 2.1 Semantic Object

语义对象是在世界中可识别、可聚焦、可建立关系的内容单位。每个对象至少应具有：

- 稳定语义身份；
- `title`；
- `orientation_summary`：是什么、为何与当前区域相关、继续进入会看到什么；
- 对象角色与推荐可见尺度；
- provenance 与置信状态；
- 与其他对象的语义关系；
- 一个或多个空间 footprint 或锚点。

短关键词可以作为远景 label 和检索 tag，但不能代替 `orientation_summary`。

### 2.2 Refinement

`refines(child, parent)` 表示下层对象解释、分解、实例化或提供证据方向给上层对象。它不是独占所有权关系。

- 一个 parent 可以被多个 child 细化。
- 一个 child 可以同时细化多个 parent。
- parent 可以跨 segment；child 不需要位于 parent 的封闭盒子内。
- refinement 应包含语义角色，例如 `component`、`mechanism`、`comparison`、`history`、`controversy`、`practice`、`evidence_direction` 或 `bridge`。
- Runtime 不得按数组下标、生成顺序或最近距离凭空推断 refinement。

### 2.3 Footprint 与 Anchor

- **Footprint** 表示一个语义区域在某尺度上的覆盖范围，可以重叠、交叉和跨越 segment 边界。
- **Anchor** 是对象用于稳定布局、聚焦和尺度过渡的位置，不等于唯一父节点。
- Cartographer 提供语义邻近、refinement、桥接关系和覆盖意图；Runtime 依据这些语义生成稳定身份、坐标、footprint、防碰撞和区块归属。

### 2.4 WorldSegment

`WorldSegment` 只负责分块生产、缓存、持久化和增量传输。它必须满足：

- segment 边界不显示为语义边界；
- 同一对象可由稳定身份跨 segment 引用或合并；
- 相邻 segment 使用已有锚点和覆盖摘要避免重复；
- segment 不能成为 L0/L1/L2 的隐藏父节点；
- segment ready 与否不改变已显示对象的既有空间位置。

## 3. 四种尺度语法

### 3.1 L0 — 领域视野（Domain Field）

回答：**这片知识世界由哪些大区域构成，我目前靠近哪里？**

典型对象：

- 领域与实践场；
- 跨领域地带；
- 主地标；
- 宏观问题；
- 大尺度未知边界。

L0 应呈现区域、方向和邻接，不应铺满同义关键词。一个 Seed 可以同时落在多个领域 footprint 的交界。

### 3.2 L1 — 主题视野（Topic Field）

回答：**当前领域内有哪些主题邻域、分支和交叉问题？**

典型对象：

- 主题与问题簇；
- 方法族与应用族；
- 历史、社会和工程分支；
- 连接两个领域的桥接主题；
- 仍待展开的主题空白。

L1 对象不要求分别隶属唯一 L0。主题可以跨领域延展，footprint 可相互重叠。

### 3.3 L2 — 解释视野（Explanation Field）

回答：**这个主题内部如何工作、由什么组成、有哪些差异、争议、实践路径与证据方向？**

典型对象：

- 组件与机制；
- 因果链与过程；
- 比较维度与权衡；
- 争议、限制和失败模式；
- 时间线、人物、组织或标准；
- 实践步骤、工具链与评估方法；
- 通向来源的 evidence direction。

L2 的正式名称是“解释视野”，不是“来源方向”。来源方向只是 L2 的一种对象角色。用户在没有打开网页时，也应能从 L2 理解这片区域为何值得探索。

### 3.4 L3 — 来源视野（Source Field）

回答：**哪些真实材料支持、说明、反驳或实例化当前解释？**

典型对象：

- 经 Scout 观察成功的网页、论文、PDF、图片或本地文档；
- 来源快照与可追溯元数据；
- 来源和 L2 解释对象之间的 supports、critiques、documents、exemplifies 等关系。

强制边界：

- Cartographer 只能提出 L3 候选，不能创建真实来源对象。
- 候选在 Scout 成功前不属于 L3 主画布。
- 失败候选进入恢复或替换状态，不显示破损瓦片。
- 同一来源可以支撑多个 L2 对象；一个 L2 对象也可以由多个来源共同支撑。

## 4. 可变密度规则

- 各尺度不得设置相同的语义对象配额，也不得要求每个 segment 恰好生成相同数量的 L0、L1 和 L2。
- 密度由语义差异性、当前覆盖缺口、用户兴趣、已有世界去重结果、证据可得性和成本预算共同决定。
- 一个成熟领域可以拥有少量稳定 L0、较多 L1 与高密度 L2；一个新兴领域也可以只有一个宽泛 L0、少量 L1 和大面积 fog。
- 没有新增语义价值时，某区域允许不产生新对象。空白必须表示为 fog 或已覆盖，不以同义改写填满配额。
- 节点大小、可见性和聚合由重要性、footprint、相机尺度和重叠程度决定，而不是单纯由 layer 决定。

## 5. 空间与尺度过渡

### 5.1 向下进入

1. 保持世界 XY 不变。
2. 使用相机中心及其附近 footprint 判断主要上下文，不要求先点击唯一父节点。
3. 上层区域退为轮廓、地标或低权重上下文；相关下层对象在其稳定位置显现。
4. 若相机位于两个上层区域交界，下层应混合呈现两者的解释对象和桥接对象。
5. 缺失内容显示 fog，并通过相机弱信号提高后台生产优先级；切层本身不得等待生成。

### 5.2 同尺度横移

- 横移是在同一语义平面中穿越 footprint，不是切换父盒子。
- 接近相邻领域或主题时，其下层对象应逐渐进入视野；既有对象不得整体重排到新的局部网格。
- 跨 chunk 后的语义身份、坐标与关系保持稳定。

### 5.3 向上收束

1. 保持 XY 不变。
2. 选择覆盖相机中心且语义权重最高的上层 footprint 作为主要返回上下文。
3. 若处于交界，可同时保留两个上层区域及桥接地标，不强行回到最初进入点。
4. 下层探索发现的新上层缺口应写回既有世界对象或其 refinement，不创建平行孤儿层。

## 6. 关系语法

除 refinement 外，至少支持以下语义关系：

| 关系 | 用途 |
|---|---|
| `adjacent_to` | 空间或认知上可顺势探索 |
| `overlaps` | 两个区域覆盖部分相同内容 |
| `bridges` | 对象连接两个或更多区域 |
| `contrasts_with` | 方法、观点或实践的对照 |
| `depends_on` | 前置概念、组件或条件 |
| `supports` | 来源或解释支持某对象 |
| `critiques` | 来源或解释质疑某对象 |
| `documents` | 来源记录对象或事件 |
| `exemplifies` | 实例化抽象主题或机制 |

关系必须携带解释或角色，不能只是无语义连线。空间相近可以辅助布局，但不能自动等价为语义相关。

## 7. 跨尺度样例 A：飞机

Seed：`飞机`

### L0 领域视野

- **航空**：飞行器、空中运行与相关产业的总体领域。
- **交通系统**：飞机作为长距离运输网络的一部分。
- **机械与动力工程**：机体、推进、材料与控制的工程基础。
- **军事与公共安全**：军用航空、空域治理、事故与安全体系。

这些 footprint 相互重叠。“航空”不是装着其他对象的盒子；例如“航空安全”同时覆盖航空、交通系统和公共安全。

### L1 主题视野

- 飞行原理
- 机体结构与材料
- 动力系统
- 飞行控制与航电
- 民航运行与机场
- 航空安全与适航
- 飞机类型与任务
- 航空史与产业

主题数量不需要与 L0 相同。“动力系统”同时细化航空与机械工程；“航空安全与适航”同时细化航空、交通和公共安全。

### L2 解释视野

- 机翼压力分布、升力与失速
- 涡扇发动机的进气、压缩、燃烧和喷气过程
- 副翼、升降舵和方向舵如何改变姿态
- 复合材料减重与维修成本的权衡
- 冗余航电、故障检测与飞行员接管
- 适航审定、维修间隔与运行限制
- 枢纽航线网络对机型选择的影响

这里存在明确的多对多关系：“复合材料权衡”细化机体结构，也影响动力效率与适航维修；“失速”同时属于飞行原理、飞行控制和航空安全。

### L3 来源视野

Scout 可以验证并呈现：监管机构适航文件、飞机或发动机制造商手册、大学空气动力学教材、事故调查报告、同行评审论文。AI 提出的 URL 在观察成功前只作为候选。

### 连续移动验收

用户从 L1“机体结构”进入 L2，向“动力系统”方向横移，应逐渐看到压气机、燃烧室和涡轮等解释对象；此时向上返回 L1，主要上下文应是“动力系统”或其交界，而不是强制回到“机体结构”。全过程不创建新 tab、子 scene 或孤儿层。

## 8. 跨尺度样例 B：FlashAttention

Seed：`FlashAttention`

### L0 领域视野

- 高效机器学习系统
- GPU 与并行计算
- 深度学习算法
- 模型训练与推理基础设施

### L1 主题视野

- Attention 算法族
- GPU 内存层级
- IO-aware kernel 优化
- 数值稳定性与精度
- 框架集成与编译
- 长上下文训练与推理
- 性能评测方法

“IO-aware kernel 优化”同时细化 GPU 并行计算、高效机器学习系统和 Attention 算法族，不应被复制成三个节点。

### L2 解释视野

- HBM 与片上 SRAM 之间的数据搬运为何成为瓶颈
- Q/K/V 分块与 tile 调度
- online softmax 如何在分块时保持正确归一化
- 重计算如何交换显存占用和计算量
- backward pass 的中间状态与 IO 路径
- causal masking、可变长度序列和稀疏模式
- PyTorch 接口、编译器选择和硬件能力约束
- wall-clock、吞吐、显存、精度与 IO 复杂度的评测差异

“online softmax”同时细化算法正确性、数值稳定性和 IO-aware 分块；“评测差异”连接性能评测、框架集成和硬件约束。这些不是固定下标父子关系。

### L3 来源视野

Scout 可以验证并呈现：原始论文及版本、官方代码仓库与文档、框架集成文档、GPU 架构资料、可复现实验或独立评测。搜索结果标题或 AI 猜测 URL 不得直接成为来源瓦片。

### 连续移动验收

用户从 L1“Attention 算法族”进入 L2 后，向“GPU 内存层级”方向横移，应从 softmax 与 attention 语义逐渐过渡到 HBM/SRAM、tile 和 kernel 调度；向上返回时主要落在“IO-aware kernel 优化”或“GPU 内存层级”，而不是原始入口。

## 9. 可验收不变量

### 9.1 数据不变量

1. 同一世界的 L0、L1、L2 对象数量允许且应当不同；不存在固定 `4/4/4` 等量契约。
2. fixture 必须至少包含一个 child 细化多个 parent、一个 parent 拥有多个 child、一个 bridge 跨两个区域。
3. 同一语义对象跨 segment 只能有一个稳定身份；相邻 segment 不得各自生成同义副本。
4. 每个可聚焦 L0–L2 对象必须有 `orientation_summary`，不得只有 title/tags。
5. AI 必须显式提供 refinement 或覆盖意图；Runtime 不得用数组位置代替语义关系。
6. L3 主画布对象必须拥有成功的 Scout observation；候选和失败记录不计为 L3 对象。

### 9.2 交互不变量

1. L0 → L1 → L2 → L3 的切换保持 camera XY；任何生成事件不得修改 view。
2. 在细尺度横移后向上收束，返回区域由当前 XY 覆盖决定，不由最初入口决定。
3. 处于两个 footprint 交界时允许混合显示，不强制选择唯一父节点。
4. 横跨 segment 边界时无整体重排、重复对象、场景切换或可见接缝。
5. 切换到尚未准备的尺度立即完成视图变化，并显示真实 fog；不阻塞等待 AI。

### 9.3 视觉与理解不变量

1. 远景可以压缩为短标签，焦点和近景必须显露摘要、角色或关系，不得始终呈现词云。
2. 上下层过渡时至少保留上层轮廓、地标或路径中的一种方向线索。
3. 区域大小与密度表达语义覆盖和重要性，不得因固定网格而使所有对象同形同权。
4. 来源对象与 AI 推断地形必须有可辨认的 provenance，不以颜色装饰掩盖状态差异。

## 10. 明确禁止的退化实现

- 每个 segment 固定生成相同数量的 L0、L1、L2。
- 用 `L0[i] → L1[i] → L2[i]` 建立父子关系。
- 切层时清空世界并重新居中排一张独立网格。
- 点击节点后创建普通缩放所需的子 tab、子 scene 或隔离盒子。
- 为填满画布生成同义关键词、重复主题或无摘要节点。
- 将 WorldSegment、屏幕象限或视觉容器解释成语义父节点。
- 将未验证 URL、搜索结果占位或 AI 生成网页节点渲染成 L3 来源瓦片。
- 在 L2 把所有内容强制解释为来源目录，丢失机制、比较、争议与实践语义。

## 11. 变更规则

- Prompt、schema、布局、缓存和设计系统必须共同满足本契约，单个模块通过测试不代表产品成立。
- “飞机”和“FlashAttention”是最低跨尺度验收 fixture；后续可以增加样例，但不得用更简单的词表样例替代。
- 对象数量、视觉样式和预取半径可以实验；连续 XY、多对多 refinement、可变密度与 L3 Scout 边界不可通过实验开关绕过。
