# SeekStar 望远镜设计体系

状态：下一阶段权威设计规范  
适用范围：App Shell、连续地图、L0–L3、AI/Scout/持久化状态与产品文案

上位约束：[产品契约](../product/PRODUCT_CONTRACT.zh.md) · [连续多尺度语义契约](../product/SEMANTIC_SCALE_CONTRACT.zh.md)  
固定验收场景：[参考流程](./REFERENCE_FLOWS.zh.md)

## 1. 设计目标

SeekStar 的中心不是聊天、搜索结果、节点列表或仪表盘，而是一张可移动、可变焦、可保留方向感的认知地图。设计必须帮助用户持续回答三件事：

1. 我在哪里；
2. 周围有什么；
3. 接近哪个方向会获得什么。

视觉吸引力服务于方向感和语义密度。均匀气泡、整齐卡片和粒子效果都不能代替地图结构。

## 2. 交互宪法

- 平移表示同一尺度上的邻近探索。
- 滚轮表示改变观察尺度，不能打开隔离子场景。
- 向下变焦时，当前位置附近的区域细化为更多对象。
- 向上变焦时，从当前 XY 所属的上层区域浮出，而不是回到最初入口。
- 相机移动只是世界生产器的弱信号；已有画布不等待逐帧 AI 请求。
- 选择、圈选和提问作用于当前地图地址，不把主界面改造成聊天流。

## 3. 四种尺度的视觉语法

| 尺度 | 主对象 | 主要问题 | 默认表达 |
| --- | --- | --- | --- |
| L0 领域视野 | Region、主 Landmark、Fog | 这片世界由哪些领地和宏观张力组成 | 不规则范围、少量强地标、方向标签 |
| L1 主题视野 | Neighborhood、Thread、交界 | 当前领地有哪些主题、问题和相邻支线 | 局部区域、路径、桥接地标 |
| L2 解释视野 | Explanation、对比、机制、Evidence Route | 这里如何工作、为何重要、怎样继续调查 | 有类型的一句话解释单元，密度渐进 |
| L3 来源视野 | Source Tile | 哪些真实材料支持、反驳或展开这些解释 | 经过验证的网页、论文、PDF、图像和文档 tile |

L0–L3 是观察尺度，不是四种独立页面。产品默认文案使用中文名称；`L0` 等编号只在状态、诊断或高级界面中作为短地址出现。

## 4. 地图对象语法

### Region

大范围语义领地。必须有范围、中心、显著度和一句 framing。Region 不是等大的圆，也不是分类菜单。

### Landmark

跨一段尺度仍需被认出的稳定地标。只有真正帮助方向感的对象才跨尺度持续出现，不能把每个节点复制到每一层。

### Thread

L1 中的问题、主题分支或探索路径。Thread 可以从一个 Region 出发，也可以桥接两个 Region。它需要显示方向或关系，而不只是名词。

### Explanation

L2 的核心对象。角色至少包括：组件、机制、因果、比较、争议、时间线、方法、实践、人物/机构和开放问题。聚焦时必须显示一句“它是什么，以及为什么值得靠近”。

### Evidence Route

从 L2 Explanation 通向真实材料的调查航向，例如论文脉络、标准、开源实现、实验室、档案或社区。它是 L2 的一种对象角色，不等于整个 L2。

### Source Tile

只有 Scout 成功观察的来源才能成为主画布 L3 tile。必须表达标题、来源类型、摘要/片段、时间与验证状态。未经验证和失败候选不占据主画布。

### Fog

真实未构建、未观察或语义尚不确定的边界。Fog 要诚实但不惊慌：它表达“可继续探索”，不表达“程序坏了”。

## 5. 渐进信息密度

同一对象按镜头距离显示不同信息，而不是所有对象同时变成长卡片。

| 状态 | 显示内容 |
| --- | --- |
| 远景/边缘 | 短标题、轮廓、来源状态 |
| 中距离 | 标题、角色、12–36 字 orientation summary、邻接方向 |
| 视野中心 | 标题、一句解释、与当前区域的关系、进入后预期、来源就绪度 |
| 选中 | 完整短摘要、关键关系、来源/推断状态与可执行操作 |

关键词可以作为远景标签和搜索索引，但不能成为聚焦对象的全部内容。

## 6. 空间与转场

- 世界坐标是权威；renderer 不得为“整齐”而全量重排已生成对象。
- 防碰撞只能进行稳定、局部、可复现的小幅调整。
- 新 chunk 写入不能移动已固定地标。
- 变焦时保留一层低透明度的父区域轮廓或关键 Landmark，直到新尺度稳定。
- 一对多细化表现为区域逐渐分裂；多对一/交叠表现为边界共享或桥接，不伪装成树。
- 关系线只在能帮助方向感时出现；默认弱化，聚焦与跨区桥接时增强。
- L3 可以使用非重叠 tile 布局，但 L0–L2 不得退化成等距网格或 dashboard。

## 7. 视觉层级

- 尺寸由显著度、语义范围和焦点共同决定，不能只由相机距离决定。
- 主地标、背景对象、边界对象必须有明显层级。
- 颜色表达领域方向；不能单独承担 generated、candidate、verified、failed 等状态。
- 来源状态同时使用形状、轮廓、徽标和文案，保证色觉差异用户可辨认。
- 圆形适合远景地标，但不能成为所有宏观对象的统一容器。
- 卡片只用于确实需要一句解释或来源材料的对象，不把整个地图变成卡片墙。

## 8. 状态与错误语法

六类状态不得混用：世界段、AI 任务、来源观察、持久化、活动视图、surface/lease。

- 邻段生成失败：在对应边界显示局部 Fog 和“重新生成此区域”。不阻挡已就绪地图。
- 来源失败：留在来源恢复队列，不显示破损 L3 tile。
- 持久化失败：明确说明本地保存失败；不能由 AI JSON 错误触发。
- 视图与 surface 错误：只影响当前活动控制面，不覆盖世界内容。
- 用户层错误文案只说明影响、保持不变的内容和下一步动作。
- Provider、JSON、token、堆栈和文件路径属于“技术详情”，默认折叠。

同一个故障只能有一个主提示。Shell 和 tab surface 不得重复显示同一 toast。

## 9. Shell 与控制面

- 活动地图只有一个视图权威。
- 右栏必须消费活动 tab surface 的真实投影、选择、layer 和来源状态；不得展示独立的默认 `New Seek` scene。
- Shell 负责窗口、左栏、tab、设置与 dock；地图控制属于活动 surface 或其受控只读镜像。
- 暂未接入活动状态的按钮不进入可见产品。
- 右栏优先显示当前地图、选择和来源；AI 对话是操作入口之一，不是默认内容中心。

## 10. 文案与语言

- 默认界面语言为简体中文。
- `SeekStar`、API、URL、PDF 等稳定专名可保留英文；普通状态和操作必须中文化。
- 动态标题必须使用明确分隔符，禁止 `AI 地图助手New Seek · L0` 一类粘连。
- 不直接向用户显示 `max_tokens`、JSON position 或异常堆栈。
- “世界段”“来源”“保存”“活动视图”等术语在全产品只保留一种中文叫法。
- 语气平静、具体，不把局部失败描述成整个工作区损坏。

## 11. 响应式与无障碍

- 1280×720 必须可完成新建、平移、变焦、选择、来源观察和设置保存。
- 1600×900 允许增加上下文，而不是单纯放大空白面板。
- 键盘焦点、选中、来源状态和错误状态必须有非颜色提示。
- 动效尊重 reduced-motion；关闭动效后仍能理解尺度变化。
- 主要文字与背景达到可读对比度；Fog 的低对比不能影响操作说明。

## 12. 禁止项

- 三层等长关键词词表。
- 所有节点等大、等色、等权。
- 每切一层就把整张地图重新居中排版。
- 用更多圆、阴影和颜色掩盖语义缺失。
- 在主画布显示未经验证或失败 URL。
- 用全屏错误阻断仍可浏览的已就绪区域。
- Shell、tab surface 和右栏显示三个不同的“当前状态”。
