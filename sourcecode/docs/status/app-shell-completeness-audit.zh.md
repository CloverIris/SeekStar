# App Shell 完成度审计

日期：2026-07-13  
范围：桌面 Shell、左侧栏、设置页；不包含世界池语义生成质量。

## 本轮已闭环

- 左栏的“新建探索”和“搜索当前地图”现在是两个明确的命令意图；回车分别创建新 Seek 或搜索当前世界。
- Starter fields 会直接创建对应 Seed，不再是假按钮。
- 文件夹可作为 tab 浏览筛选器；空文件夹会给出明确空状态。
- 未实现的 Brush 已从工具栏移除，避免把未来能力伪装成当前能力。
- 设置页改为工作区、连接、智能、系统四组；草稿有“未保存 / 保存 / 放弃”明确生命周期。

## 仍未完成，但边界清楚

| 区域 | 当前缺口 | 应归属 |
| --- | --- | --- |
| 新建探索 | 空输入时只有命令焦点，没有 Seed 建议或最近轨迹 | Shell / onboarding |

| 设置生效 | 大部分设置保存后在下一次任务生效；没有统一的“即时 / 下次任务 / 重启”标记 | 设置契约 |
| 操作反馈 | 创建 Seed、刷新 tab、清缓存等缺少统一 toast / 任务反馈 | Shell 反馈层 |

## 周期边界

1. **Shell 周期**：用户意图 → tab / folder / tool / command 状态 → 可见反馈。不得同步等待 AI。
2. **世界池周期**：相机信号 → chunk 调度 → 缓存快照 → 渲染消费。不得由左栏按钮直接操控布局。
3. **来源周期**：候选 → Scout 观察 → 已验证 tile 或恢复队列。不得因设置或 UI 快捷操作绕过验证。
4. **设置周期**：草稿 → 保存 → 按生效范围应用。设置页只编辑配置，不持有世界生成或 tab 生命周期。

## 下一轮建议顺序

1. 增加 Shell 统一 toast / 后台任务反馈，并为收藏、置顶、文件夹提供快速视图。
2. 将设置页拆成 `navigation`、`workspace`、`connections`、`intelligence`、`system` 五个独立组件；为每项设置标记生效时机。
3. 建立跨 tab 的本地搜索与最近探索轨迹。
4. 在真实用户测试中验证新建、横移、滚轮、返回、保存发现这条核心路径。
