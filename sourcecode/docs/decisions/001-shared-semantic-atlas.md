# ADR 001：采用共享语义地图金字塔

状态：Accepted for next product iteration  
日期：2026-07-19

规范化结果：[连续多尺度语义契约](../product/SEMANTIC_SCALE_CONTRACT.zh.md)

## 问题

当前原型把每个世界段生产为近似等长的 L0、L1、L2 词表，再按数组下标建立父子关系。它验证了世界池、缓存和视图隔离，却没有表达真实的多尺度语义连续性。

## 决策

SeekStar 采用共享 XY 世界中的多分辨率语义场：

- `WorldSegment` 是生产、缓存和持久化信封，不是语义父节点。
- Region、Thread、Explanation 和 Source 在同一世界坐标中具有稳定身份。
- 跨尺度细化允许一对多、多对一和交叠。
- 只有少量 Landmark 跨尺度保持持续身份；不把所有对象复制到每层。
- AI 提供语义身份、短解释、细化与桥接关系；Runtime 提供 ID、稳定几何、chunk、校验和持久化。
- L3 只包含 Scout 验证成功的真实材料。

## 拒绝方案

### 每层独立词云

实现简单，但层间只是替换标题和样式，横向移动没有可解释的空间含义。

### 严格树形盒子

父子关系清楚，但横移到相邻领域后向上返回容易回到原入口，制造平行子场景。

### 每个对象四层复制

保留身份但造成固定 1:1 映射，无法表达一个区域的可变细化、交界和证据数量。

## 后果

- 生成契约需要可变数量对象、短 brief 和显式语义关系。
- 布局器不能全量覆盖持久世界坐标。
- 设计系统必须表达 Region footprint、交界、父层残影和渐进信息密度。
- 测试必须验证语义去重、可变基数、多父关系和横移返回，而不仅是 layer/XY 数值不变。
- 现有运行时代码暂不在本次文档重构中修改；实施另开受控纵切。
